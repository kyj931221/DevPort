'DevPort' 포트폴리오 프로젝트 설명서 (Spatial UI)

1. 프로젝트 개요

이 프로젝트는 개인 포트폴리오를 독창적인 '안드로이드 XR 공간 UI' 테마로 구현한 인터랙티브 웹사이트입니다. 3차원 공간에 떠 있는 듯한 반투명 패널과 동적인 배경을 통해 깊이감과 미래지향적인 사용자 경험을 제공합니다. SPA(Single Page Application) 구조를 적용하여 페이지 전환 없이 부드러운 상호작용이 가능합니다.

  • 대문 페이지 (`index.html`): 공간에 부유하는 환영 패널을 통해 포트폴리오로 입장하는 경험을 제공합니다.
  • 메인 페이지 (`main.html`): 동적인 공간 배경 위에 떠 있는 좌측 네비게이션 독(Dock)과 메인 콘텐츠 패널로 구성된 핵심 페이지입니다.

2. 주요 파일 구조 및 역할

프로젝트는 크게 **① 뼈대(Shell) 파일**, **② 콘텐츠 조각(Fragment) 파일**, **③ 기능 및 디자인 파일**로 나뉩니다.

콘텐츠 조각(Fragment) 파일 리스트

아래는 메인 페이지(`main.html`)의 동적 콘텐츠 영역에 불러오는 모든 '콘텐츠 조각' 파일의 목록입니다.

파일 이름 설명

content-table.html '콘텐츠 목차' 페이지의 내용입니다. 제작한 프로젝트들을 표 형태로 요약하여 보여줍니다.
game-cards.html '콘텐츠' 페이지의 내용입니다. 각 프로젝트를 이미지와 설명이 포함된 카드 형태로 보여줍니다.
license-award.html '수상 및 자격' 페이지의 내용입니다. 취득한 자격증과 수상 내역을 카드 형태로 보여줍니다.
device.html '보유 장비' 페이지의 내용입니다. 개발에 사용하는 하드웨어 장비 목록을 보여줍니다.
side-bar/sidebar.html 메인 페이지의 좌측 네비게이션 독(Dock)을 구성하는 메뉴 아이템들의 목록입니다.
manual-devport.html 현재 보고 있는 이 'DevPort 프로젝트 설명서'의 내용입니다.
manual-another-project.html 다른 프로젝트에 대한 설명서 예시 파일입니다.

3. HTML-CSS-JS 연결 흐름 (동작 원리)

사용자가 사이트에 접속해서 메뉴를 클릭하기까지의 과정은 아래와 같습니다.

  1. 최초 접속 (`index.html`): 사용자는 사이트 주소로 접속하여 `index.html`을 가장 먼저 봅니다. 브라우저는 `spatial-welcome.css`를 불러와 공간 UI 스타일의 환영 패널을 표시합니다.
  2. 입장하기 (`main.html` 로딩): 사용자가 '입장하기' 버튼을 클릭하면 `main.html` 페이지로 이동합니다.
  3. 스크립트 실행 (`scripts.js`): `main.html`과 연결된 `spatial-ui.css`와 `scripts.js`가 실행됩니다. `scripts.js`는 `sidebar.html`의 내용을 읽어와 좌측 네비게이션 독을 구성하고, 초기 콘텐츠로 `my-info.html`을 메인 패널에 표시합니다.
  4. 메뉴 클릭 및 콘텐츠 교체: 사용자가 좌측 네비게이션 독의 아이콘을 클릭하면, `scripts.js`의 `loadContent` 함수가 호출되어 페이지 새로고침 없이 메인 패널의 내용만 해당 HTML 조각 파일로 교체합니다.
